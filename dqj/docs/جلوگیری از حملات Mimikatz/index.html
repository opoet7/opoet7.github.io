جلوگیری از حملات Mimikatz

ابزار Mimikatz یکی از ابزار های اصلی تیم های قرمز برای استخراج رمز های عبور و هش ها و ... می باشد این ابزار برای اجرا نیاز به یک کاربر با سطح دسترسی نظیر Local Administrator دارد. در این مقاله راه های جلوگیری از حملات این ابزار را بررسی خواهیم کرد.

دسترسی Debug​

این ابزار برای اجرا نیاز به دسترسی Debug دارد به صورت پیش فرض Local Administrator شامل این دسترسی می باشد این دسترسی از جمله دسترسی هایی است که برای دیباگ کردن Process ها و کرنل مورد استفاده قرار می گیرد برای غیر فعال کردن این دسترسی از مسیر زیر در Group Policy استفاده کنید و کاربرانی که نیاز به این دسترسی را دارند در این مسیر وارد کنید.(این دسترسی خیلی کم استفاده می شود و یکی از موارد استفاده آن توسط برنامه نویسان سیستمی می باشد)

Group Policy Management Editor -> Windows Settings -> Security Settings -> Local Policies -> User Rights Assignment -> Debug programs -> Define these policy settings:

با اعمال این تنظیمات همان طور که در تصویر زیر مشخص است دسترسی SeDebugPrivilege غیر فعال شده است.

پروتکل WDigest​

این پروتکل برای اولین با در ویندوز XP برای احراز هویت توسط پروتکل HTTP طراحی و پیاده سازی شد با توجه به کارکرد این پروتکل رمز های عبور به واسطه فعال بودن این پروتکل به صورت plain-text در LSASS ذخیره می شود ابزار Mimikatz می تواند توسط دستور زیر رمز عبور های را از LSASS به صورت plain-text استخراج کند.

sekurlsa::wdigest

برای غیر فعال کردن این پروتکل می توانید از مسیر رجیستری زیر استفاده کنید.

HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\SecurityProviders\WDigest

و مقادیر کلید های Negotiate و UseLogonCredential را برابر 0 قرار دهید.

پس از اعمال تنظیمات بالا با اجرای مجدد دستور sekurlsa::wdigest با خطای زیر مواجه خواهید شد.

محافظت از LSA​

اعتبار سنجی کاربران از جمله وظایف LSASS می باشد مایکروسافت از ویندوز ۸.۱ به بعد قابلیتی برای محافظت از LSASS برای جلوگیری از خواندن حافظه و یا تزریق کد به پروسس LSASS ارایه کرده است. نفوذگران ممکن است با اجرای دستور زیر اقدام به استخراج رمز های عبور از حافظه اصلی مربوط به پروسس LSASS کنند.

sekurlsa::logonPasswords

برای فعال سازی قابلیت LSA Protection از مسیر رجیستری زیر کلید RunAsPPL را با مقدار ۱ ایجاد نمایید.

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\LSA

با اعمال تنظیم بالا با اجرای مجدد دستور sekurlsa::logonPasswords با خطای زیر مواجه خواهید شد.

Credential Caching​

در صورتی که دامین کنترلر در دسترسی نباشد ویندوز سعی می کند تا توسط آخرین هش رمزهای عبور کش شده در سیستم احراز هویت کند این هش ها در مسیر رجیستری زیر نگهداری می شوند.

HKEY_LOCAL_MACHINE\SECURITY\Cache

ابزار Mimikatz می تواند با استفاده از دستور lsadump::cache به این هش ها دسترسی پیدا کند.

ویندوز به صورت پیش فرض ۱۰ هش رمز عبور را در این کش نگهداری می کند برای غیر فعال کردن این قابلیت و جلوگیری از دزدیدن هش ها با استفاده از Group Policy مقدار مسیر زیر را به 0 تغییر دهید.

Computer Configuration -> Windows Settings -> Local Policy -> Security Options -> Interactive Logon: Number of previous logons to cache -> 0

با اعمال تنظیم بالا با اجرای دستور lsadump::cache با خطای زیر مواجه خواهید شد.

گروه Protected Users​

مایکروسافت از ویندوز سرور ۲۰۱۲ قابلیتی به نام Protected Users را معرفی کرده که به واسطه آن می توان از کاربران با دسترسی بالا در مقابل دزدی رمز عبور جلوگیری کرد و اعضای این گروه فقط می توانند از طریق پروتکل Kerberos احراز هویت کنند.

اعضای این گروه به صورت خودکار از پالیسی های مربوط به Kerberos که در تصویر زیر مشخص هست تبعیت می کنند.

با استفاده از دستور زیر می توان یک حساب کاربری را به عضویت این گروه درآورد.

Add-ADGroupMember –Identity 'Protected Users' –Members Jane

غیر فعال کردن ذخیره سازی رمز های عبور به صورت Plain Text​

برای غیر فعال کردن ذخیره سازی رمز های عبور به صورت Plain Text توسط Group Policy در مسیر زیر گزینه Store Passwords using reversible encryption را Disabled کنید.

Computer Configuration -> Security Settings -> Account Policies -> Password Policy
