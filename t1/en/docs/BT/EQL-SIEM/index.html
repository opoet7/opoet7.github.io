Event Query Language - EQL

EQL​

Event Query Language (EQL) is a query language for event-based time series data, such as logs, metrics, and traces. EQL helps blueteam quickly find suspicious activity and improve detections of attacker techniques defined in MITRE’s ATT&CKTM.

Advantage of EQL​

  • EQL lets you express relationships between events.
    Many query languages allow you to match single events. EQL lets you match a sequence of events across different event categories and time spans.
  • EQL has a low learning curve.
    EQL syntax looks like other common query languages, such as SQL. EQL lets you write and read queries intuitively, which makes for quick, iterative searching.
  • EQL is designed for security use cases.
    While you can use it for any event-based data, we created EQL for threat hunting. EQL not only supports indicator of compromise (IOC) searches but can describe activity that goes beyond IOCs.

Event​

Event Kind​

gives high-level information about what type of information the event contains,without being specific to the contents of the event

  • alert
  • enrichment
  • event
  • metric

Event Category​

Indicates the second level in the ECS category hierarchy

  • process

  • network

  • file

  • registry

  • host

Event Type​

represents a categorization "sub-bucket" that, when used along with the event.category field values

  • protocol

  • allowed

  • connection

  • start

  • user

event.type:connection AND event.type:allowed

event.category:process AND event.type:start

Event Outcome​

simply denotes whether the event represents a success or a failure from the perspective of the entity that produced the event.

  • failure
  • success
  • unknown

event.category:file AND event.type:create AND event.outcome:success

Basic EQL query​

search in process event category for specific process

GET /*/_eql/search
{
  "query":"""
    process where process.name == "hh.exe"
    """
}

search in any event category

GET /*/_eql/search
{
  "query":"""
   any where network.protocol == "http"
    """
}

search with not operator

GET /*/_eql/search
{
 "query": """
        process where process.name == "cmd.exe" and process.parent.name != "explorer.exe"
  """
}

search process command line not null

GET /*/_eql/search
{
  "query": """
    process where process.name == "regsvr32.exe" and process.command_line != null
  """
}

search multi process 

GET /*/_eql/search
{
  "query": """
    process where process.name in~ ("cmd.exe", "powershell.exe", "pwsh.exe")
  """
}

search sequence in two event category 

GET /winlogbeat-8.2.0/_eql/search
{
 "query": """
    sequence
      [ process where process.name == "cmd.exe" ]
      [ file where stringContains(file.name, "bcrypt.dll") ]
  """
}

matching sequence to a timespan

GET /*/_eql/search
{
"query": """
    sequence with maxspan=1h
      [ process where process.name == "regsvr32.exe" ]
      [ file where stringContains(file.name, "scrobj.dll") ]
  """
}

Match events that share the same field values

GET /*/_eql/search
{
  "query": """
    sequence with maxspan=20h
      [ process where process.name == "cmd.exe" ] by process.pid
      [ file where stringContains(file.name, "test.dll") ] by process.pid
  """
}
GET /*/_eql/search
{
  "query": """
   sequence by process.pid with maxspan=1h
       [ process where process.name == "regsvr32.exe" ]
       [ file where stringContains(file.name, "scrobj.dll") ]
  """
}

match events with a process.args_count value of 4

GET /*/_eql/search
{
 "query": """
      process where ( 4 / process.args_count ) == 1
  """
}
GET /*/_eql/search
{
  "query": """
  sequence by user.name with maxspan=10h
  [file where file.extension == "dll" ]
  [ process where true ]

  """
}

process has network connection

GET /winlogbeat-8.2.0/_eql/search
{
  "query": """
      sequence by process.pid
      [process where process.name == "svchost.exe"]
      [network where true]
  """
}

Event hists

GET /winlogbeat-8.2.0/_eql/search?filter_path=-hits.events    
{
  "query": """
    any where process.name == "regsvr32.exe"                
  """,
  "size": 200                                             
}
